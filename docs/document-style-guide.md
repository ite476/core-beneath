## 목적
모든 문서를 처음 읽는 사람도 이해하기 쉽게, 그리고 팀 전체가 일관되게 작성하도록
작성 규칙을 정의합니다.

## 범위
문서 언어 정책, 공통 메타 필드, 섹션 순서, 링크 규칙, 문체 규칙을 다룹니다.

## 본문
### 문서 언어 정책
- 기본 본문 언어는 한국어를 사용합니다.
- 핵심 문서(예: `README.md`, `docs/game-overview.md`, `docs/mvp-scope.md`)에는
  영어 요약 섹션을 제공합니다.
- 고유 명사, 기술 용어는 처음 등장 시 한국어 설명과 함께 표기합니다.

### 필수 메타 필드
모든 일반 문서는 아래 메타 필드를 포함해야 합니다.
- `Status`
- `Last Updated`
- `Owner`
- `Source of Truth`
- `Related Docs`

### 필수 섹션 순서
모든 일반 문서는 아래 순서를 유지합니다.
1. `목적`
2. `범위`
3. `본문`
4. `결정사항`
5. `미해결 이슈`
6. `변경 이력`

### 문체 및 가독성 규칙
- 독자가 처음 읽는다고 가정하고 배경을 짧게 제공합니다.
- 한 문단은 하나의 의도를 설명합니다.
- 표/목록을 우선 사용하여 스캔 가능성을 높입니다.
- "왜 필요한가"와 "언제 갱신하는가"를 함께 씁니다.

### 링크 및 GitHub 렌더링 규칙
- 문서 링크는 상대 경로를 사용합니다.
- 문서 내 점프는 헤딩 앵커 기반으로 작성합니다.
- 링크가 있는 문서는 PR에서 반드시 링크 검사를 통과해야 합니다.

### 예외 문서
- `.github/PULL_REQUEST_TEMPLATE.md` 같은 폼 문서는 구조 목적상
  필수 섹션 순서 예외를 허용합니다.

### 문서 수정 트리거
- 공통 메타 필드 정의가 바뀔 때
- 필수 섹션 순서가 바뀔 때
- 언어/문체/링크 규칙이 바뀔 때

## 결정사항
- 문서 스타일은 "초심자 친화 + 일관성 + 유지보수성"을 최우선으로 합니다.
- 공통 메타 필드와 섹션 순서를 표준 계약으로 운영합니다.
