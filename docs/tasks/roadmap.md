## 목적
MVP 달성을 위한 단계별 마일스톤과 산출물을 정의합니다.

## 범위
문서/개발의 중기 진행 계획을 다룹니다. 주간 상세 작업은 현재 스프린트 문서에서
관리합니다.

## 본문
### 마일스톤 계획
| 단계 | 목표 | 주요 산출물 | 상태 |
| --- | --- | --- | --- |
| Phase 0: Documentation Baseline | 문서 체계 정비 | 문서 허브, 운영 정책, 코어 설계 문서 | 완료 |
| Phase 1: Unity Bootstrap | 개발 환경 및 기본 구조 고정 | Unity 프로젝트 초기화, 폴더 구조, 기초 씬 | 예정 |
| Phase 2: Core Loop Vertical Slice | 채굴-생산-물류-전력 기본 루프 구현 | 플레이 가능한 핵심 루프 | 예정 |
| Phase 3: Progression + End Goal | 기술 단계/코어 채굴/점프드라이브 | 엔딩 가능한 MVP | 예정 |
| Phase 4: Polish + Validation | 안정화/튜닝/문서 회귀 점검 | MVP 릴리스 후보 | 예정 |

### 우선순위 원칙
1. 루프 완결성 > 콘텐츠 양
2. 시스템 명확성 > 복잡한 확장성
3. 문서 최신성 > 임시 구두 합의

### 문서 수정 트리거
- 마일스톤 순서나 목표가 바뀔 때
- 각 Phase 상태가 바뀔 때
- MVP 완료 기준이 바뀔 때

## 결정사항

- 다음 단계는 Unity Bootstrap(Phase 1)으로 진행합니다.
