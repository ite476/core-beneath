## 목적
개발 시작 단계에서 반드시 완료해야 할 작업을 체크리스트로 고정합니다.

## 범위
Unity 프로젝트 부팅, 최소 시스템 뼈대, 문서 동기화 검증 항목을 다룹니다.

## 본문
### 환경/부팅
- [ ] Unity LTS 에디터 설치 및 프로젝트 생성
- [ ] 에디터 버전 정보 확인(`ProjectVersion.txt`)
- [ ] 기본 씬 실행 확인

### 구조/규약
- [ ] [project-structure.md](project-structure.md) 기준 폴더 구조 반영
- [ ] 기본 네임스페이스 전략 확정
- [ ] 코딩 표준 문서 공유 및 팀 합의

### 핵심 시스템 착수
- [ ] Grid 배치 기초 컴포넌트 생성
- [ ] 자원 노드/수동 채굴 프로토타입
- [ ] 생산 건물 1종(Miner 또는 Smelter) 동작 검증

### 문서/운영 검증
- [ ] 변경된 구현 항목이 관련 문서에 반영되었는지 확인
- [ ] PR 템플릿 문서 체크 항목 완료
- [ ] 문서 품질 CI 게이트 통과

### 완료 기준
- 문서만 보고 신규 개발자가 프로젝트를 실행하고 첫 작업을 시작할 수 있다.
- 최소 1개의 자동화 경로(채굴 -> 처리)가 플레이 가능 상태로 동작한다.

### 문서 수정 트리거
- 체크리스트 항목 추가/삭제
- 완료 기준 변경
- 스프린트 목표 변경

## 결정사항
- 첫 스프린트는 "부팅 가능성 + 최소 루프 검증"을 핵심 성공 기준으로 삼습니다.
