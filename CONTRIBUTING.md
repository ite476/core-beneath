## 목적
이 문서는 저장소 기여자의 공통 작업 절차를 정의합니다. 코드와 문서의 변경이
항상 일관되고 검증 가능한 형태로 반영되도록 하는 것이 목적입니다.

## 범위
브랜치/커밋/PR 기준, 문서 동기화 규칙, 리뷰 준비 항목을 다룹니다.

## 본문
### 작업 절차
1. 관련 문서를 먼저 읽고 변경 범위를 정의합니다.
2. 변경을 구현하고 문서 영향도를 함께 반영합니다.
3. PR 템플릿 항목을 모두 점검합니다.
4. 문서 품질 CI(마크다운 린트, 링크 검사)를 통과시킵니다.

### 브랜치와 커밋 권장 규칙
- 브랜치명: `feature/...`, `fix/...`, `docs/...`
- 커밋 메시지: 변경 의도를 드러내는 한 줄 제목 + 필요 시 본문
- 하나의 PR에는 하나의 일관된 목적을 담습니다.

### 커밋 컨벤션 (필수)
- 커밋 제목은 외부 노출 기준으로 **한글**을 사용합니다.
- 커밋 본문도 가능하면 **한글**로 작성합니다.
- 제목 형식: ``타입: 요약`` 권장
- 권장 타입: `문서`, `기능`, `수정`, `리팩터`, `테스트`, `설정`, `CI`
- 예시:
  - `문서: 저장소 입구 및 문서 허브 구조 정비`
  - `CI: 문서 품질 게이트 워크플로 추가`

### 이슈/PR 분류 규칙
- 라벨/마일스톤 운영 규칙은 [docs/repository-workflow.md](docs/repository-workflow.md)를 따릅니다.
- PR 생성 시 `유형`, `영역`, `상태`, `우선순위` 라벨을 최소 1개씩 적용합니다.
- PR은 반드시 마일스톤 1개를 지정합니다.

### 문서 동기화 필수 규칙
- 기능/설계/범위가 바뀌면 관련 문서를 같은 PR에서 갱신합니다.
- 문서를 수정하면 `Last Updated`, `Related Docs`, `변경 이력`을 갱신합니다.
- 문서 링크는 상대 경로를 사용하고 깨진 링크를 남기지 않습니다.

### PR 준비 체크
- [ ] 변경 목적과 범위가 PR 본문에 명확히 설명되어 있다.
- [ ] 문서 영향이 있는 경우 연관 문서를 함께 수정했다.
- [ ] `docs/README.md`의 문서 맵과 링크가 필요한 경우 함께 갱신했다.
- [ ] CI 문서 품질 게이트를 통과했다.

### 문서 수정 트리거
- 기여 프로세스, 리뷰 기준, PR 정책이 바뀔 때
- 문서 품질 게이트 정책이 바뀔 때
- 브랜치/커밋 규칙이 바뀔 때

## 결정사항
- 코드 변경과 문서 변경은 가능한 한 동일 PR에서 반영합니다.
- PR 템플릿을 기준으로 문서 영향 점검을 필수화합니다.
- 커밋 제목/본문은 한국어 우선 원칙을 적용합니다.

## 미해결 이슈
- 향후 코드 스타일 자동 포맷터 도입 시 커밋/리뷰 규칙 보강 필요
